What is this website used for?

The author of this application often encounters errors related to:

  • Consistency in the code
  • Non-functional error
  • Naming Convention
  • etc.

This is also a place to note tips, personal standards of what "good" code is like.

Comparing code with this list before committing code can be considered a way of self-checking the quality of your code, thereby allowing coders to focus more on thinking logically without having to worry too much about minor bugs in the code.

UI and ideas inspired by Angular Checklist.
